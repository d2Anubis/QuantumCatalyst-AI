The Industrial Problem

Catalyst discovery is broken at industrial scale

Industrial biogas operations convert organic waste into CBG, biomethane, and green hydrogen. The bottleneck is catalytic:

  • 6–18 months per lead catalyst in traditional high-throughput screening
  • DFT errors in transition-metal systems — exactly where the best catalysts live
  • Broken feedback loop — lab results never reach the computational model
  • No reasoning layer — engineers get scores, not explanations
  • Fragmented institutional knowledge — every failed experiment is lost

Biogas upgrading

H₂S poisons amine solvents. No rapid sorbent replacement screening tool exists.

Biomethane → Green Hydrogen

Ni-based SMR catalysts coke above 750°C. Better promoters are needed at scale.

Sabatier CO₂ Methanation

Ni/Al₂O₃ thermal management unsolved at scale. Runaway risk in fixed-bed reactors.

The real bottleneck in catalyst discovery is not lack of data — it's the inability to efficiently convert experimental outcomes into continuously improving scientific intelligence.

Our Solution

An AI-native scientific discovery
operating system

QuantumCatalyst AI combines quantum energy profiling, generative candidate design, and a closed-loop learning architecture into a single platform that field and R&D engineers can operate — no quantum chemistry PhD required.

Layer 1

Quantum Energy Profiling

VQE-inspired simulation evaluates transition-metal catalyst descriptors with higher electron-correlation fidelity than classical DFT. Drop-in replaceable with real IBM Quantum / AWS Braket backend — zero UI changes.

Layer 2

Generative AI Design

Diffusion-inspired generative step proposes 6 novel alloy variants per run by mutating top performers in the latent descriptor space — exploring beyond the known chemical space.

Layer 3

Continuous Learning

Bayesian bias-update loop learns from every lab result submitted. Each experiment shifts activity, selectivity, and stability predictions closer to reality — permanently.

  • Step 1–2: Engineer selects a biogas-pathway reaction (Biogas Upgrading, SMR, Sabatier, Waste-to-Biogas). Conditions auto-populate from seeded reaction templates.
  • Step 3: VQE energy profiler computes ground-state energies, activation barriers (C–C formation, H-transfer, O-removal), and per-candidate confidence scores.
  • Step 4: Top database catalysts seed a generative step — 6 novel alloy variants per run, with novelty scores tracking how far they deviate from known space.
  • Step 5–6: Pareto ranking across activity / selectivity / stability. Lab results feed back into the model via online Bayesian bias update — each run is smarter.

Technical Architecture

Built to be upgraded, not rewritten

Current tech stack

Backend

Node.js + Express — pipeline orchestration, dual-provider LLM routing (OpenAI + Gemini), Bayesian feedback model

Frontend

Vanilla HTML/CSS/JS (zero build step for the app) + Next.js 15 marketing page. 3Dmol.js for molecular viewer.

AI

OpenAI GPT-4o / Gemini 1.5–2.0 Flash. Auto-detect from key prefix. Deterministic rule-based fallback — always works offline.

Quantum

VQE abstraction layer with identical API contract to Qiskit Runtime. Drop-in real quantum backend with zero frontend changes.

Upgrade path to production

  • Real quantum backend: swap runVqeLikeEnergyProfile() for Qiskit Runtime call — zero UI changes
  • Persistent model: replace in-memory feedback object with PostgreSQL + Prisma — model survives server restarts
  • Live SCADA integration: connect Pipeline Input to plant sensors — auto-populate real feedstock composition
  • Multi-user auth: add NextAuth.js — individual experiment histories per engineer
  • Real DFT data: plug in Open Catalyst Project (1.2M DFT calculations) as the retrieval backbone
